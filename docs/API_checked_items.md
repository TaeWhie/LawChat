## 2. 지능형 법령정보지식베이스 (JSON만)

| 항목 | 비고 |
|------|------|
| 법령용어 JSON | 용어 정의·설명. 중복 시 **하단(지능형)** 사용 (연계 데이터 포함) |
| 일상용어 JSON | 사용자 표현 |
| 관련법령 JSON | 조문–다른 법령 연관 |
| 지능형 법령검색 시스템 검색 API JSON | 키워드/조문 검색 |
| 지능형 법령검색 시스템 연관법령 API JSON | 연관법령 조회 |
| 법령용어-일상용어 연계 JSON | 법령용어 → 일상 표현 |
| 일상용어-법령용어 연계 JSON | 일상 표현 → 법령용어 (이슈 정규화) |
| 법령용어-조문 JSON | 용어 ↔ 조문 |
| 조문-법령용어 JSON | 조문 ↔ 용어 |

## 3. API별 활용 방안

### 3.1 법령·행정규칙

| API | 활용 방식 |
|-----|-----------|
| 대한민국 현행법령 | 근로기준법·고용보험법·산재보험법 등 **노동 관련 법률 원문** 수집·동기화. 목록으로 대상 법령 선정 후 본문 조회 → 조 단위 파싱 → 벡터 스토어 또는 로컬 MD 갱신. RAG 검색의 **기본 법령 출처**. |
| 현행 행정규칙 | 근로기준법 시행령·시행규칙 등 **시행령·시행규칙 원문** 수집. 법률과 동일하게 조(條) 단위로 파싱 후 스토어에 반영. 법률만으로 부족한 세부 기준·산식 보강용. |
| 법령 별표·서식 | 법률 부칙의 **별표**(퇴직금 산정표, 최저임금 업종표 등)·**서식**(신청서·제출 서식) 수집. "어떤 서식으로 제출하나요?", "별표 제x호에 따르면 …" 등 답변·인용 시 활용. |
| 행정규칙 별표·서식 | **시행령·시행규칙** 부칙의 별표·서식 수집. 법령 별표·서식과 동일하게 세부 산정표·신청 서식 안내·인용 시 활용. |

### 3.2 판례·헌재·해석·심판

| API | 활용 방식 |
|-----|-----------|
| 판례 | 사용자 상황·이슈와 **유사한 판례** 검색(검색어·참조조문 활용). 판시사항·판결요지 요약을 RAG 컨텍스트에 넣거나 "참고 판례"로 인용. `precedent_query`로 쿼리 생성 후 목록/본문 조회. |
| 헌재결정례 | 헌법·기본권 관련 노동 이슈(예: 근로3권)가 있을 때 **헌재 입장** 인용. 노동 도메인 필터 후 상위 N건만 컨텍스트에 포함. |
| 법령해석례 | 부처 통합 **법령 해석례** 검색. 이슈·조문과 맞는 해석만 골라 RAG에 "해석례에 따르면 …" 형태로 보강. |
| 행정심판례 | 행정심판 사례가 이슈와 맞을 때 **심판례 요지** 인용. 노동 관련만 필터해 보조 근거로 사용. |

### 3.3 위원회·부처 법령해석

| API | 활용 방식 |
|-----|-----------|
| 개인정보보호위원회 | 근로자 개인정보·감시·동의 등 **개인정보 이슈**가 노동 상담에 섞일 때 결정례·해석 인용. |
| 고용보험심사위원회 | **고용보험**(실업급여·육아휴직·취업촉진 등) 관련 심사 사례 검색. 이슈가 고용보험일 때 요지만 RAG에 포함. |
| 공정거래위원회 | 노동과 겹치는 **불공정행위·하도급** 등 이슈 시 위원회 결정·해석 참고. |
| 노동위원회 | **부당해고·임금·해고예고** 등 구제 신청 관련 **노동위 결정례** 검색. "노동위원회 결정에 따르면 …" 형태로 답변에 반영. |
| 산업재해보상보험 재심사위원회 | **산재 인정·보상** 관련 재심사 사례. 산재 이슈일 때 요지 인용. |
| 고용노동부 법령해석 | 노동 법령 **공식 해석** 1순위. 이슈·조문 확정 후 관련 해석 검색 → 요지를 RAG 컨텍스트에 넣어 "고용노동부 해석에 따르면 …" 인용. |
| 법제처 법령해석 | 고용노동부 외 **통합·다부처** 해석. 노동 관련만 필터해 고용노동부 해석과 함께 보강 인용. |
| 법무부 법령해석 | 노동과 겹치는 **민사(손해배상·계약)·형사(업무상 과실 등)·소송 절차** 등 이슈 시 해석 인용. "민사·형사·소송 관련하여 법무부 해석에 따르면 …" 형태로 보조. |

### 3.4 지능형 법령정보지식베이스

| API | 활용 방식 |
|-----|-----------|
| 법령용어 JSON | **용어 정의·설명** 조회. "임금이란?" 등 용어 질문 시 공식 정의 반환 또는 RAG 프롬프트에 용어 설명 보강. |
| 일상용어 JSON | 사용자 표현 수집·분석. (연계 JSON과 함께 쓰는 경우 많음.) |
| 관련법령 JSON | 특정 조문 조회 시 **연관된 다른 법령·조문** 목록 확보. "관련하여 ○○법 제△△조도 참고할 수 있습니다" 등 답변 확장. |
| 지능형 법령검색 시스템 검색 API | 이슈/키워드로 **공식 검색** 1회 수행. 벡터 검색 결과와 병합하거나 상위 N건만 보조로 사용. |
| 지능형 법령검색 시스템 연관법령 API | 조문 단위 **연관법령** 공식 조회. 관련법령 JSON과 유사하나 API 호출로 실시간 반영. |
| 법령용어-일상용어 연계 JSON | **법령 용어 → 일상 표현** 매핑. 답변 문구를 "일반인이 쓰는 말"로 보완할 때 참고. |
| 일상용어-법령용어 연계 JSON | **사용자 말 → 법령 용어** 매핑. 이슈 정규화(예: "월급"→"임금") 시 기존 `ISSUE_TO_CATEGORY`·분류 JSON과 함께 사용하거나 대체. |
| 법령용어-조문 JSON | 용어로 **해당 조문** 찾기. "평균임금" 검색 시 관련 조문 목록 확보. |
| 조문-법령용어 JSON | 조문으로 **해당 법령용어** 찾기. 조문 설명·요약 생성 시 용어 리스트 참고. |

### 3.5 흐름 요약

1. **입력** → 일상용어-법령용어 연계·검색 API로 이슈/용어 정규화·검색 후보 확보.  
2. **법령** → 현행법령·행정규칙으로 확보한 원문이 RAG 기본 검색 대상.  
3. **해석·판례·위원회** → 이슈·조문에 맞는 해석례·판례·위원회 결정을 검색해 답변에 "해석/판례/결정에 따르면 …" 형태로 인용.  
4. **용어·연관** → 법령용어·관련법령·연계 JSON으로 용어 설명·연관 조문 보강.

## 4. API 신청 전 최종 점검

| 확인 항목 | 내용 |
|-----------|------|
| **요청 포맷** | 목록/본문 모두 **JSON**으로 신청. XML·HTML만 제공되는 항목(예: 별표·서식 본문)은 제공처 안내에 따름. |
| **법령용어 중복** | 상단(대분류) 법령용어 + 지능형 법령용어 둘 다 신청해 둔 상태. 연동 시 중복이면 **지능형(하단)** 기준 사용 → `docs/TODO_verify_law_terms.md` 참고. |
| **미포함** | 현행 학칙공단, 조약, 영문법령, 중소벤처기업부 법령해석, 현행 자치법규 등은 **의도적으로 미포함**(노동 도메인·우선순위 기준). 필요 시 추후 추가. |
| **신청 시 확인할 것** | ① 인증 방식(API 키 등) ② 레이트 리밋 ③ 이용약관(상업 이용·출처 표기) ④ 제공처에서 사용하는 **정확한 API/데이터셋 이름** (목록 JSON·본문 JSON 표기 확인). |

## 5. 참고

- **법령용어** 상단(대분류) vs 하단(지능형): 동일 여부는 나중에 API 문서로 확인. 중복이면 **하단(지능형)** 기준 사용 → `docs/TODO_verify_law_terms.md` 참고.
- XML은 사용하지 않음 (JSON만 체크).
- 체크했는데 목록에 없는 항목 있으면 알려주세요.
